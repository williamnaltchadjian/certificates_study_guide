## O que é IA Generativa (GenAI)?

Diferente da IA Preditiva (que classifica ou prevê números), a GenAI cria **novos dados** (texto, imagem, áudio, código) baseando-se em padrões aprendidos.

### Modelos de Base (Foundation Models - FMs)

São modelos treinados em uma escala massiva de dados (terabytes de internet/livros) que podem ser adaptados para várias tarefas sem precisar de treino do zero.

## 📌 Arquitetura Transformer

| Tema | Conceito | O que lembrar para a prova | Exemplos / Observações |
|------|----------|---------------------------|------------------------|
| **Texto → Números** | Tokens | Unidade básica de processamento do texto | Palavra pode ser dividida (ex: In + crível) |
|  | Embeddings | Representação numérica do significado | Palavras semelhantes têm vetores próximos ("rei" / "rainha") |
|  | Vetores | Coordenadas matemáticas do significado | “Endereço” do conceito no espaço vetorial |
| **Arquitetura** | Transformer | Arquitetura base dos LLMs modernos | Usada por GPT, Claude, Llama |
|  | Self-Attention | Permite entender contexto e relação entre palavras distantes | Diferencial dos Transformers |
|  | Foundation Models | Modelos gigantes pré-treinados que podem ser adaptados | Não precisam ser treinados do zero |
| **Tipos de Modelos** | Multimodal | Processa múltiplos formatos (texto, imagem, áudio) | Ex: enviar imagem e pedir descrição |
|  | Difusão (Diffusion Models) | Principal técnica de geração de imagens | Adiciona ruído e aprende a revertê-lo |
| **Prompt Engineering** | Zero-shot | Tarefa sem exemplos | “Traduza: Oi” |
|  | Few-shot | Tarefa com alguns exemplos prévios | Modelo aprende pelo padrão |
|  | Chain-of-Thought (CoT) | Solicita raciocínio passo a passo | Reduz erros lógicos |

## 📌 Ciclo de Vida de Modelos de IA Generativa – Resumo em Tabela

| Etapa | O que acontece | Pontos-chave para prova |
|-------|---------------|------------------------|
| **Seleção e Preparação de Dados (Data Selection & Preparation)** | Coleta massiva de dados (livros, sites, códigos, artigos) | Dados são limpos, deduplicados e filtrados (remoção de conteúdo tóxico) |
| **Seleção de Modelos (Model Selection)** | Escolha do modelo ideal | Critérios: custo, janela de contexto (context window), modalidade (texto/imagem), latência |
| **Pré-treinamento (Pre-training)** | Treinamento para prever o próximo token | Aprende conhecimento geral, mas não segue instruções |
| **Ajuste Fino e Alinhamento (Fine-Tuning & Alignment)** | Especialização e alinhamento do modelo | Inclui ajuste de instrução (Instruction Fine-Tuning) e RLHF (Reinforcement Learning from Human Feedback) |
| **Avaliação (Evaluation)** | Testes antes da produção | Métricas: acurácia (accuracy), toxicidade (toxicity), robustez (robustness) |
| **Implantação e Inferência (Deployment & Inference)** | Modelo disponibilizado via API (endpoint) | Otimização para custo e velocidade |
| **Monitoramento e Feedback (Monitoring & Feedback)** | Coleta de logs e feedback dos usuários | Pode gerar nova rodada de ajuste fino |

|--------|---------------|----------|

| Ajuste Fino (Fine-Tuning) | Especializar em um domínio específico | Faculdade ou especialização |
| RAG (Retrieval-Augmented Generation) | Consultar um manual antes de responder | Prova com consulta ao livro |
| Avaliação (Evaluation) | Testar segurança e precisão | Exame final (vestibular) |

## 🎯 Dica de Prova

- **Fine-Tuning:** altera os pesos do modelo (model weights) para especializá-lo.  
- **RAG (Retrieval-Augmented Generation):** não altera o modelo; apenas adiciona informações externas no momento da inferência.

## 📌 Vantagens da IA Generativa (Generative AI) – Resumo em Tabela

| Vantagem | O que é | Pontos-chave para prova | Comparação com ML Tradicional (Traditional ML) |
|-----------|--------|-------------------------|--------------------------------|
| **Adaptabilidade (Adaptability / Versatility)** | Um único Modelo de Base (Foundation Model) executa múltiplas tarefas | Mesmo modelo pode resumir (summarization), traduzir (translation) e gerar código (code generation) | ML tradicional exige um modelo diferente para cada tarefa |
| **Capacidade de Resposta e Agilidade (Responsiveness & Agility)** | Criação rápida de soluções usando Prompts | Redução drástica de time-to-market; uso de Few-shot prompting | Antes exigia meses de coleta e rotulagem de dados (data labeling) |
| **Simplicidade (Simplicity / Democratization)** | Interface baseada em linguagem natural (Natural Language Interface) | Não exige programação; analistas podem interagir diretamente | ML tradicional exige cientistas de dados |
| **Eficiência de Dados (Data Efficiency)** | Menor dependência de dados rotulados | Usa auto-supervisão (Self-supervised Learning) e transfer learning (via RAG ou Fine-Tuning) | ML tradicional depende fortemente de rotulagem manual |
| **Inovação e Criatividade (Innovation & Creativity)** | Capacidade de gerar novas ideias e conteúdos | Atua como “copiloto” (Copilot) criativo | Sistemas tradicionais apenas analisam dados históricos |

## ⚠️ Riscos e Limitações da IA Generativa (Generative AI) – Resumo em Tabela

| Desvantagem | O que é | Causa | Mitigação |
|-------------|--------|-------|------------|
| **Alucinações (Hallucinations)** | Geração de informações falsas, porém convincentes | Modelo é um preditor estatístico de palavras, não um banco de fatos | RAG (Retrieval-Augmented Generation) / Knowledge Bases |
| **Não Determinismo (Non-determinism)** | Respostas diferentes para o mesmo prompt | Natureza probabilística do modelo | Reduzir Temperatura (Temperature = 0) |
| **Falta de Interpretabilidade (Lack of Interpretability / Black Box)** | Difícil explicar por que o modelo tomou uma decisão | Bilhões de parâmetros (model parameters) sem regra explícita | Ferramentas como SageMaker Clarify |
| **Imprecisão e Falta de Raciocínio Lógico (Imprecision / Lack of True Reasoning)** | Erros matemáticos ou falhas em cadeias lógicas longas | Modelo trabalha por associação de padrões (pattern recognition) | Chain-of-Thought (CoT) prompting |
| **Viés e Toxicidade (Bias & Toxicity)** | Respostas ofensivas, estereotipadas ou discriminatórias | Dados de treinamento herdados da internet | Guardrails (ex: Amazon Bedrock Guardrails) |

## 💰 Modelos de Preço e Trade-offs no Amazon Bedrock – Resumo em Tabela

| Tema | Opção | Vantagem | Compensação (Trade-off) | Quando usar na prova |
|------|--------|----------|--------------------------|----------------------|
| **Modelo de Preço (Pricing Model)** | **On-Demand (Token-based Pricing)** | Paga apenas pelos tokens processados (input + output tokens) | Sem garantia de throughput (vazão) em picos | Volumes baixos ou imprevisíveis |
|  | **Provisioned Throughput** | Desempenho garantido (guaranteed throughput) e latência estável (stable latency) | Custo fixo mais alto mesmo sem uso total | Produção com tráfego constante e previsível |
| **Tipo de Modelo (Model Type)** | **Modelos de Base (Foundation Models – FMs)** | Baixa barreira de entrada e custo inicial menor | Sem personalização específica do negócio | Uso rápido via API |
|  | **Modelos Personalizados (Fine-Tuned Models)** | Especialização com dados próprios | Custo de treinamento (GPU hours), armazenamento e geralmente Provisioned Throughput | Casos que exigem domínio específico |
| **Tamanho do Modelo (Model Size)** | **Modelos Grandes (Large Models)** | Maior precisão e raciocínio complexo | Mais caros por token e maior latência | Tarefas técnicas ou críticas |
|  | **Modelos Pequenos (Small / Efficient Models)** | Mais rápidos e baratos | Menor capacidade de raciocínio | Tarefas simples e alto volume |
| **Infraestrutura e Região (Infrastructure & Region)** | **Multi-AZ (Multi–Availability Zone)** | Alta disponibilidade (high availability) | Maior custo de infraestrutura | Sistemas críticos |
|  | **Uso entre Regiões (Cross-Region Usage)** | Acesso a mais modelos ou preços melhores | Custo de transferência de dados (data transfer cost) e impacto em residência de dados (data residency) | Quando região local não tem o modelo desejado |

## 🎯 Critérios para Seleção de Modelos (Model Selection) – Resumo em Tabela

| Critério | Opções | Quando escolher | Trade-off |
|-----------|--------|----------------|-----------|
| **Modalidade (Modality)** | Text-to-Text | Resumos (summarization), extração de entidades (entity extraction) | Apenas texto |
|  | Text-to-Image | Geração de imagens (image generation), marketing, design | Não processa lógica textual complexa |
|  | Multimodal | Entende texto + imagem (text + image understanding) | Pode ter custo maior |
| **Latência vs. Complexidade (Latency vs. Complexity)** | Modelos Pequenos (Small Models – ex: 8B) | Resposta rápida (low latency), tarefas simples | Menor capacidade de raciocínio |
|  | Modelos Grandes (Large Models – ex: 70B) | Raciocínio complexo (complex reasoning), coding, escrita longa | Maior custo e latência |
| **Janela de Contexto (Context Window)** | Pequena | Frases curtas e parágrafos | Pode “esquecer” início de textos longos |
|  | Grande | Análise de documentos extensos (long documents) | Pode custar mais |
| **Multilíngue (Multilingual Support)** | Suporte limitado | Aplicações focadas em inglês | Pode gerar erros gramaticais em outros idiomas |
|  | Suporte robusto a NLP | Aplicações em português ou mercados globais | Critério importante para qualidade |
| **Custo (Token-Based Pricing)** | Modelo leve (Lightweight Model) | Alto volume e tarefas repetitivas | Menor capacidade técnica |
|  | Modelo avançado (Advanced Model) | Tarefas complexas e esporádicas | Maior custo por milhão de tokens |
| **Personalização (Customization)** | Sem Fine-Tuning | Uso genérico | Sem especialização |
|  | Com Fine-Tuning / Continued Pre-training | Domínios técnicos (jurídico, médico) | Custo adicional de treinamento |

## 🧠 Mapa Mental de Decisão (Decision Matrix)

| Critério | Escolha Modelo Menor/Simples se... | Escolha Modelo Maior/Complexo se... |
|------------|------------------------------------|------------------------------------|
| **Latência (Latency)** | Precisa resposta em tempo real (real-time chat) | Precisão é mais importante que velocidade |
| **Custo (Cost)** | Volume de requisições é muito alto | Tarefa é esporádica e muito complexa |
| **Tokens / Context Window** | Entradas curtas | Análise de múltiplos documentos longos |
| **Tipo de Tarefa (Task Type)** | Classificação (classification), extração (extraction), tradução (translation) | Raciocínio jurídico, escrita criativa, lógica complexa |

---

## 🎯 Dica de Prova

Se a questão mencionar **custo-benefício para tarefa simples e repetitiva**, escolha:

➡ **Modelo menor (Small / Efficient Model)**  
➡ Preço baseado em **Token-Based Pricing**

Se mencionar **análise complexa ou documentos extensos**, escolha:

➡ **Modelo maior (Large Model)**  
➡ Janela de contexto grande (Large Context Window)




## 🎯 Dica de Ouro para a Prova

- **Tráfego constante + previsibilidade de custos → Provisioned Throughput**
- **Teste ou volume imprevisível → On-Demand (Preço por Token)**

## Check-list de Termos para Decorar

- [ ] **Zero-shot Prompting:** Pedir algo sem dar exemplos
- [ ] **Few-shot Prompting:** Dar 2 ou 3 exemplos no prompt antes da pergunta
- [ ] **Chain-of-Thought:** Pedir para o modelo "pensar passo a passo"
- [ ] **Negative Prompt:** (Em imagens) Dizer o que você não quer que apareça
