# AWS Documentation: Segurança, Conformidade e Governança para Soluções de IA

Este documento centraliza o conhecimento sobre segurança, conformidade e governança para soluções de IA na AWS, cobrindo **14% do conteúdo pontuado** na certificação AWS AI Practitioner.

---

## Introdução

Esta seção aborda como garantir que soluções de IA sejam:

* Seguras por design
* Conformes com leis e regulações
* Governadas de forma responsável

!!! important
    Este domínio representa 14% da prova. Foco em IAM, KMS, CloudTrail, SageMaker Clarify e Macie.

---

## Segurança em IA na AWS

### Conceitos fundamentais:

| Assunto | O que você precisa saber |
| :--- | :--- |
| **Modelo de Responsabilidade Compartilhada** | AWS protege a infraestrutura; você protege dados, acessos e configurações |
| **IAM (Identity and Access Management)** | Controle de acesso com usuários, roles e políticas |
| **Princípio do menor privilégio** | Conceder apenas o acesso estritamente necessário |
| **Criptografia** | Dados em trânsito (TLS) e em repouso (KMS) |
| **Monitoramento** | Logs e auditoria com CloudTrail |

### Boas práticas de segurança:
* Usar IAM roles em vez de credenciais fixas
* Habilitar MFA (Multi-Factor Authentication)
* Criptografar dados com KMS
* Monitorar atividades com CloudTrail e GuardDuty

---

## Conformidade (Compliance)

Compliance garante que o uso de IA:

* Siga leis e regulações vigentes
* Respeite a LGPD / GDPR
* Proteja dados pessoais e sensíveis
* Mantenha registros auditáveis

| Assunto | O que estudar |
| :--- | :--- |
| **LGPD / GDPR** | Proteção de dados pessoais |
| **PII (Personally Identifiable Information)** | Identificação e proteção de dados sensíveis |
| **Data Residency** | Onde os dados estão armazenados |
| **Certificações AWS** | ISO, SOC, HIPAA, etc. |

!!! note
    O **AWS Artifact** é a ferramenta para acessar relatórios de conformidade e auditorias da AWS.

---

## Governança de IA

Governança de IA significa definir políticas, processos e responsabilidades claras para uso ético dos modelos.

| Assunto | O que entender |
| :--- | :--- |
| **IA Responsável** | Justiça, transparência, explicabilidade |
| **Bias em modelos** | Modelos podem gerar decisões injustas |
| **Explainability** | Entender por que o modelo tomou uma decisão |
| **Human-in-the-loop** | Revisão humana quando necessário |
| **Model Monitoring** | Detectar drift e problemas em produção |

## Resumo Estratégico para Prova

### O que mais cai na prova:
* Modelo de responsabilidade compartilhada
* Proteção de dados sensíveis (PII)
* Viés (bias) e IA responsável
* Explicabilidade de modelos
* Criptografia em trânsito e em repouso
* Controle de acesso com IAM

### Serviços essenciais para o AI Practitioner:
* **IAM** – Controle de acesso
* **KMS** – Criptografia
* **CloudTrail** – Auditoria
* **SageMaker Clarify** – Bias e Explicabilidade
* **A2I** – Revisão humana
* **Macie** – Detecção de PII

!!! tip
    Memorize a regra rápida: Acesso → IAM | Criptografia → KMS | Auditoria → CloudTrail | Bias → Clarify | Revisão humana → A2I | PII → Macie

## Pegadinhas comuns

| Mito | Realidade |
| :--- | :--- |
| **Segurança** | ≠ apenas criptografia — envolve IAM, monitoramento e isolamento |
| **Compliance** | ≠ governança — são complementares, não sinônimos |
| **Bias** | ≠ hallucination — bias é discriminação; hallucination é dado inventado |
| **A2I** | ≠ SageMaker Clarify — A2I é revisão humana; Clarify detecta bias |
