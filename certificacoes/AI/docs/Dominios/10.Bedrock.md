# 1. Amazon Bedrock — Overview

O **Amazon Bedrock** é um serviço totalmente gerenciado da AWS que permite **construir aplicações de IA generativa** utilizando **Foundation Models (FMs)** de diferentes provedores através de **uma única API**, sem necessidade de gerenciar infraestrutura.


## Comparação entre os modelos Amazon Nova

| Modelo | Performance | Custo | Casos de uso |
|---|---|---|---|
| Nova Micro | Baixa | Muito baixo | Chatbots simples, classificação |
| Nova Lite | Médio | Baixo | Resumos, assistentes virtuais |
| Nova Pro | Alto | Médio | Copilots, análise complexa |
| Nova Premier | Muito alto | Alto | IA avançada e decisões estratégicas |



## Comparação dos modelos Titan

| Modelo | Tipo | Principais usos |
|---|---|---|
| Titan Text | Processamento de linguagem natural | Chatbots, geração de texto, resumo |
| Titan Embeddings | Representação vetorial | Busca semântica, RAG, recomendações |
| Titan Image Generator | Geração de imagens | Marketing, design, criação visual |







## . Amazon Bedrock — Foundation Model (FM)

Os **Foundation Models (FMs)** são modelos de IA de grande escala treinados com enormes volumes de dados e capazes de realizar várias tarefas.


# 3. Amazon Bedrock — Fine-Tuning

O **Fine-Tuning** permite **adaptar um modelo base para um caso específico**, treinando o modelo com dados personalizados.

## Quando usar Fine-Tuning

- adaptar o modelo ao **tom da empresa**
- melhorar respostas em **domínios específicos**
- personalizar comportamento do modelo

## Processo de Fine-Tuning

1. Selecionar um Foundation Model
2. Preparar um dataset personalizado
3. Treinar o modelo no Bedrock
4. Implantar o modelo ajustado

## Benefícios

- maior precisão
- respostas mais relevantes
- adaptação ao contexto do negócio

## Ponto importante para certificação

**Fine-Tuning modifica o modelo.**

Diferente do **RAG**, que apenas fornece dados externos para o modelo.

---

# 4. Amazon Bedrock — FM Evaluation

O **FM Evaluation** permite **comparar e testar diferentes modelos antes da implementação**.

Isso ajuda a escolher o modelo com melhor desempenho para uma aplicação específica.

## Tipos de avaliação

### Avaliação automática

Usa métricas automatizadas para avaliar:

- qualidade da resposta
- relevância
- consistência

### Avaliação humana

Usuários avaliam:

- clareza
- utilidade
- precisão
- segurança

## Benefícios

- escolher o melhor modelo
- comparar prompts
- reduzir custos de experimentação

# 5. Amazon Bedrock — RAG & Knowledge Base

## RAG (Retrieval Augmented Generation)

O **RAG** combina:

- recuperação de informações
- geração de texto com IA

Isso permite que o modelo responda utilizando **dados atualizados da empresa**.

### Fluxo do RAG
Usuário faz pergunta
↓
Sistema busca dados relevantes
↓
Dados são enviados ao modelo
↓
Modelo gera resposta baseada nos dados


## Vantagens

- respostas mais precisas
- uso de dados atualizados
- não precisa treinar o modelo novamente

---

## Knowledge Base for Bedrock

A **Knowledge Base** permite conectar documentos da empresa ao modelo para uso com RAG.

### Fontes de dados suportadas

- Amazon S3
- SharePoint
- Confluence
- Websites
- Documentos corporativos

## Processo

1. ingestão dos documentos
2. criação de **embeddings**
3. armazenamento em banco vetorial
4. recuperação de dados durante consultas

---

## Bancos vetoriais suportados

- Amazon OpenSearch
- Pinecone
- Redis
- Aurora PostgreSQL (pgvector)

# 6. Amazon Bedrock — Guardrails

Os **Guardrails** são mecanismos de **segurança e controle de conteúdo**.

Eles ajudam a evitar respostas inadequadas ou perigosas geradas pela IA.

## Objetivos

- prevenir conteúdo ofensivo
- proteger dados sensíveis
- garantir conformidade com políticas

---

## Tipos de proteção

### Content Filtering

Bloqueia conteúdos como:

- violência
- conteúdo sexual
- linguagem tóxica

---

### Sensitive Information Filtering

Detecta e bloqueia dados como:

- CPF
- números de cartão de crédito
- dados pessoais

---

### Topic Control

Permite definir assuntos permitidos ou proibidos.

Exemplo:

Permitido:
- suporte ao cliente

Bloqueado:
- política

---

### Word Filtering

Permite bloquear palavras específicas.

---

# 7. Amazon Bedrock — Guardrails (Hands-On)

Fluxo de uso dos Guardrails:

1. Criar um Guardrail
2. Definir políticas de conteúdo
3. Configurar filtros de PII
4. Associar o Guardrail ao modelo

### Fluxo de execução
Entrada do usuário
↓
Validação do Guardrail
↓
Modelo gera resposta
↓
Guardrail valida resposta
↓
Resposta enviada ao usuário


Isso garante **proteção tanto no input quanto no output**.

# 8. Amazon Bedrock — Agents

Os **Agents for Bedrock** permitem criar aplicações onde a IA **executa tarefas automaticamente usando APIs ou serviços externos**.

## Funcionamento

O agente:

1. entende o pedido do usuário
2. decide quais ações executar
3. chama APIs externas
4. retorna a resposta final

---

## Exemplo

Usuário pede:

"Reserve um hotel para amanhã."

O agente pode:

- consultar API de hotéis
- verificar disponibilidade
- realizar reserva

---

## Integrações comuns

- AWS Lambda
- APIs REST
- bancos de dados
- sistemas internos

---

## Componentes de um Agent

- Foundation Model
- Action Groups
- APIs ou Lambda
- Knowledge Base

---

# 9. Amazon Bedrock — CloudWatch Integration

O Bedrock se integra ao **Amazon CloudWatch** para monitoramento e observabilidade.

## Métricas monitoradas

- número de requisições
- latência
- erros
- tokens processados

## Logs

Podem ser enviados para:

- CloudWatch Logs
- Amazon S3

## Benefícios

- monitoramento de performance
- identificação de falhas
- controle de custos

# 10. Amazon Bedrock — Pricing

O Bedrock utiliza o modelo **pay-as-you-go**.

Ou seja, você paga apenas pelo uso.

## Principais fatores de custo

### Tokens processados

Cobrança por:

- tokens de entrada
- tokens de saída

---

### Fine-Tuning

Cobrança pelo treinamento do modelo personalizado.

---

### Hospedagem de modelos

Modelos customizados podem gerar custo de hospedagem.

---

### Knowledge Base

Custos associados a:

- armazenamento
- processamento
- banco vetorial

---

## Ponto importante para certificação

O Bedrock **não cobra infraestrutura**, apenas **uso dos modelos**.

- tarefas simples

---

### Nova Lite

- equilíbrio entre custo e performance

---

### Nova Pro

- maior capacidade de raciocínio
- tarefas mais complexas

---

### Nova Premier

- modelo mais avançado
- aplicações complexas de IA

---

## Capacidades do Amazon Nova

- geração de texto
- raciocínio avançado
- aplicações multimodais
- automação de tarefas
- criação de agentes
